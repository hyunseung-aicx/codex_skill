# Learning Journal Skill

AI가 대신 해결해주는 환경에서도 개발자의 지식과 사고력이 성장하도록,
학습 인사이트를 체계적으로 기록하고 누적한다.

## Purpose

Claude Code가 에러를 고치고, 코드를 짜주고, 설계를 해줄 때
"왜 그렇게 했는지"를 개발자가 이해하고 내재화하도록 돕는다.

핵심 차별점 (dev-journal과의 차이):
- dev-journal: **프로젝트** 관점 (뭘 했는가, 커밋, PR, 메트릭)
- learning-journal: **개발자** 관점 (뭘 배웠는가, 왜 그런가, 다음엔 어떻게)

## When to Use

- 세션 종료 시 "오늘 배운 것 정리해줘"
- 복잡한 에러 해결 후 "이거 정리해줘"
- 새로운 기술/패턴 도입 후
- 주간 학습 회고 시
- `/learning-journal` 명시적 호출 시

## 학습 기록 유형

### 1. Error Resolution Note (에러 해결 노트)

에러를 해결할 때마다 작성:

```markdown
## [날짜] 에러: {에러 제목}

**증상**: 무엇이 발생했는가
**에러 메시지**: `실제 에러 메시지`

**근본 원인 (WHY 체인)**:
1. 왜? → {직접 원인}
2. 왜? → {더 깊은 원인}
3. 왜? → {근본 원인}

**해결**: 어떻게 고쳤는가 (코드 1~3줄)
**원리**: 왜 이 해결법이 작동하는가 (1~2문장)
**방지법**: 같은 실수를 어떻게 예방하는가

**난이도**: 초급 / 중급 / 고급
**카테고리**: #runtime #type #async #config #dependency #network
```

### 2. Pattern Discovery (패턴 발견)

새로운 패턴이나 기법을 배울 때:

```markdown
## [날짜] 패턴: {패턴 이름}

**한 줄 정의**: {패턴}은 {목적}을 위한 {방법}이다.

**언제 쓰는가**: {조건 1~2개}
**언제 안 쓰는가**: {반대 조건}

**핵심 코드**:
```{lang}
// 최소 예시 (5줄 이내)
```

**대안**: {대안 패턴} — {왜 이번엔 안 골랐는지}
**트레이드오프**: {장점} vs {단점}

**카테고리**: #design-pattern #react #django #python #javascript
```

### 3. Concept Map (개념 연결)

연관된 개념들의 관계를 정리:

```markdown
## [날짜] 개념: {주제}

**핵심 요약**: 2~3문장

**관련 개념**:
- {개념A} → {개념B}: {관계 설명}
- {개념B} → {개념C}: {관계 설명}

**내가 헷갈렸던 부분**: {혼동 포인트}
**명확해진 이해**: {올바른 이해}

**카테고리**: #concept #fundamentals
```

### 4. Decision Log (설계 결정 학습)

왜 이 설계를 선택했는지 기록:

```markdown
## [날짜] 결정: {무엇을 결정했는가}

**상황**: 어떤 문제를 풀어야 했는가
**선택지**:
1. {옵션A}: {장점} / {단점}
2. {옵션B}: {장점} / {단점}

**선택**: {옵션X}
**이유**: {왜 이걸 골랐는가 — 1~2문장}
**배운 점**: {이 결정에서 배운 원칙}

**카테고리**: #architecture #decision
```

## 출력 위치

```
docs/learning/
├── YYYY-MM-DD.md          # 일일 학습 기록
├── weekly/
│   └── YYYY-WNN.md        # 주간 학습 요약
└── index.md               # 카테고리별 인덱스 (자동 생성)
```

프로젝트에 `docs/learning/` 이 없으면 자동 생성.
Git 커밋하지 않음 (개인 학습 기록이므로 선택적).

## 자동 트리거 조건

다음 상황에서 학습 기록 작성을 제안:
1. **에러 해결 완료**: "이 에러 해결 과정을 학습 노트로 정리할까요?"
2. **새 패턴 첫 사용**: "이 패턴을 처음 사용했네요. 기록할까요?"
3. **세션 종료 시** (Stop hook 연동): 핵심 학습 1~2줄 요약

## 주간 학습 요약

`/learning-journal --weekly` 호출 시:

```markdown
# 주간 학습 요약 (YYYY-MM-DD ~ YYYY-MM-DD)

## 이번 주 배운 것 TOP 3
1. {가장 임팩트 큰 학습}
2. {두 번째}
3. {세 번째}

## 에러 해결 ({N}건)
- {에러1}: {근본 원인 한 줄} → {방지법}
- {에러2}: ...

## 새로 익힌 패턴 ({N}건)
- {패턴1}: {한 줄 정의}
- {패턴2}: ...

## 다음 주 학습 목표
- [ ] {아직 이해 부족한 영역}
- [ ] {더 깊이 파고 싶은 주제}

## 성장 지표
- 에러 독립 해결률: {N}% (Claude 설명 전에 원인 추측 성공)
- 새 패턴 적용: {N}건
- 반복 에러: {N}건 (줄어들수록 좋음)
```

## 사용 예시

```bash
# 오늘 학습 기록 생성
/learning-journal

# 특정 에러 해결 기록
/learning-journal "CORS 에러 해결"

# 주간 학습 요약
/learning-journal --weekly

# 특정 카테고리 학습 이력
/learning-journal --search "#async"
```

## 핵심 원칙

1. **간결하게**: 각 기록은 읽는 데 1분 이내
2. **WHY 중심**: "뭘 했는가"가 아니라 "왜 그런가"
3. **반복 활용**: 같은 에러 재발 시 이전 기록 참조
4. **노이즈 최소화**: 이미 아는 것은 기록하지 않음
5. **누적 효과**: 주간 요약으로 지식 고도화
